ข้อเสนอโครงการพัฒนาระบบ e-Tracking Backend

MNRE-SLC-012

ระบบ e-Tracking Backend

1. หลักการและเหตุผล

ในปัจจุบัน การปรับปรุง (update) ข้อมูลสถานะของคำขอข้อมูลข่าวสารในระบบติดตามคำขอข้อมูลข่าวสารของกระทรวงทรัพยากรธรรมชาติและสิ่งแวดล้อม (e-Tracking) ยังใช้วิธีให้ผู้พัฒนาระบบบริการอิเล็กทรอนิกส์ (Frontend Developer) ดำเนินการแก้ไข source code ของฐานข้อมูลสถานะคำขอ ซึ่งต้องใช้ความรอบคอบและระมัดระวังอย่างยิ่ง หากให้เจ้าหน้าที่อื่น เป็นผู้ดำเนินการแทน อาจไปลบหรือเติมรหัสคอมพิวเตอร์คลาดเคลื่อนได้โดยง่าย หากมีรหัสผิดเพียงอักขระใดอักขระหนึ่ง จะทำให้ระบบ e-Tracking เสียหายทันที ดังนั้น เพื่อป้องกันความเสียหายดังกล่าว กับเพื่ออำนวยความสะดวกในการปรับปรุงฐานข้อมูลสถานะคำขอโดยไม่ต้องอาศัยความรู้ความชำนาญด้าน Coding อันจะทำให้ผู้ใช้งานระบบ e-Tracking ได้รับประสบการณ์ที่ดี สำนักงานปลัดกระทรวงทรัพยากรธรรมชาติและสิ่งแวดล้อม จึงจัดทำโครงการพัฒนาระบบสนับสนุนการปรับปรุงฐานข้อมูลของระบบติดตามสถานะคำขอข้อมูลข่าวสารของกระทรวงทรัพยากรธรรมชาติและสิ่งแวดล้อม (e-Tracking Backend) เพื่อให้มีระบบ Backend สำหรับใช้ควบคุมการแก้ไข source code ของฐานข้อมูลของระบบ e-Tracking โดยไม่ต้องเข้าถึงเครื่องแม่ขายจากระยะไกล (Remote Access) แต่อย่างใด จะทำให้การ update ฐานข้อมูลสถานะคำขอข้อมูลข่าวสารของกระทรวงทรัพยากรธรรมชาติและสิ่งแวดล้อมเป็นไปอย่างสะดวก รวดเร็ว และเรียบร้อย


2. วัตถุประสงค์ของโครงการ

เพื่อพัฒนาระบบสนับสนุนการปรับปรุงฐานข้อมูลของระบบติดตามสถานะคำขอข้อมูลข่าวสารของกระทรวงทรัพยากรธรรมชาติและสิ่งแวดล้อม (e-Tracking Backend)


3. วิธีดำเนินงาน

การพัฒนาระบบ e-Tracking Backend เริ่มจากพิจารณาศึกษา source code ของระบบ e-Tracking แล้วออกแบบ/เขียนโปรแกรมเพื่อรับข้อมูลหมายเลขประจำตัวประชาชน (13 หลัก) ของผู้ขอ และทำการเข้ารหัสข้อมูลดังกล่าวตามมาตรฐานสากล เป็นการคุ้มครองข้อมูลส่วนบุคคลของผู้ขอข้อมูล รวมถึงรับข้อมูลสถานะคำขอเป็นข้อความ (text) โดยมีกระบวนการตรวจสอบความถูกต้องของข้อมูลนำเข้า (input validation) แล้วเริ่มกระบวนการ Backend โดยแปลงข้อความที่กรอก เป็นรหัสฝั่งแม่ข่าย (server-side script) ในภาษา PHP แล้วออกคำสั่งเรียกดูรหัสข้อมูลทั้งหมดในฐานข้อมูลของระบบ e-Tracking แล้ว ดำเนินการตัดแต่งรหัสดังกล่าว ตามข้อมูลนำเข้าที่เข้ารหัสไว้ แล้วต่อเติมกลับเป็นฐานข้อมูลรูปแบบเดิม เมื่อระบบได้ดำเนินการ update จนเสร็จสิ้นกระบวนการแล้ว ให้มีการแจ้งเตือนผู้ใช้งานทราบต่อไป ทั้งนี้ สามารถสรุปกรอบแนวคิดโครงสร้างการทำงานของระบบหลังบ้าน (Backend Operation System) ตาม sequence diagram ด้านล่างนี้
